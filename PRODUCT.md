# Product

## Register

product

## Users

Developers who use Start as a desktop coding assistant and need local, remote, and mobile access to stay easy to configure without pulling them out of their workflow.

## Product Purpose

Start helps developers work with a coding assistant from desktop-first surfaces, with supporting relay and web/mobile connection flows. Success means common setup, settings, and session tasks feel direct, fast, and trustworthy.

## Brand Personality

Clear, focused, and capable. The interface should feel calm enough for long work sessions while still carrying a recognizable product identity.

## Anti-references

Avoid generic SaaS decoration, heavy marketing-style panels inside task surfaces, vague theme names, over-rounded controls, duplicate styling hooks, and UI that obscures diagnostics or assistant output.

## Design Principles

- Keep setup surfaces concise and task-first.
- Preserve transparency and system UI behavior in the Electron shell.
- Use restrained product styling with brand moments only where they improve feedback or recognition.
- Prefer durable, testable implementation over one-off visual patches.
- Keep conversational content direct and place secondary metadata behind progressive disclosure.

## Accessibility & Inclusion

Support reduced motion, stable layouts, keyboard focus states, and readable contrast in both light and dark themes. Motion should convey state and degrade cleanly when users prefer reduced motion.
